¿ QUE ES UN CONTROL DE VERSIONES ?

"Un control de versiones es un sistema que permite gestionar y registrar los cambios realizados en archivos y directorios a lo largo del tiempo. Es fundamental en el desarrollo de software para mantener un historial de modificaciones, coordinar el trabajo entre varios desarrolladores y revertir a versiones anteriores si es necesario"


¿QUE ES GIT?

"Git es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar y rastrear cambios en el código fuente de un proyecto. A diferencia de los sistemas de control de versiones centralizados, Git permite a cada desarrollador tener una copia completa del repositorio en su propia máquina, lo que facilita el trabajo en equipos distribuidos y la gestión de cambios."

¿ QUE ES UN REPOSITORIO?

"Un repositorio es un lugar donde se almacena y organiza el código fuente y otros archivos relacionados con un proyecto. Existen dos tipos principales de repositorios:"


LOCALES

-Son repositorios que existen en la máquina del usuario. Cada usuario puede tener su propio repositorio local, donde puede realizar cambios y hacer pruebas de manera independiente.

REMOTOS

-Son repositorios que están alojados en servidores en la red (como GitHub, GitLab, Bitbucket). Los repositorios remotos sirven para compartir cambios entre varios usuarios y equipos.

¿QUE ES GITHUB?

"GitHub es una plataforma de alojamiento de código fuente basada en la web que utiliza Git como sistema de control de versiones. Permite a los desarrolladores almacenar sus repositorios en la nube, colaborar en proyectos, realizar revisiones de código y gestionar problemas y solicitudes de cambio. Ofrece funcionalidades adicionales como integración continua, documentación y herramientas para gestionar proyectos."

¿ CUALES SON LOS ESTADOS EN GIT?

"En Git, los archivos pueden estar en uno de tres estados:"


MODIFICADO

El archivo ha sido cambiado, pero esos cambios aún no se han registrado en el sistema de control de versiones.

PREPARADO (STAGED)

El archivo ha sido marcado para incluirse en el próximo commit, pero aún no se ha registrado.

CONFIRMADO (COMMITTED)

El archivo ha sido registrado en el repositorio y los cambios han sido almacenados en el historial de commits

¿QUE ES UNA RAMA EN GIT Y PARA QUE SIRVE?

"Una rama en Git es una línea independiente de desarrollo que permite trabajar en diferentes versiones del proyecto simultáneamente. Las ramas se utilizan para:"

- Desarrollar nuevas características sin afectar la rama principal (generalmente llamada main o master).
- Corregir errores sin interrumpir el desarrollo principal.
- Realizar experimentos y pruebas de manera aislada."
